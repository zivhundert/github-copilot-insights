## Implementation Steps

### Step 1: Fix `metricsCalculator.ts`

- Replace the line-ratio acceptance rate with event-based:
  ```
  acceptanceRate = SUM(code_acceptance_activity_count) / SUM(code_generation_activity_count) × 100
  ```
- Add `aiCodeAmplification` metric: `totalLocAdded / totalLocSuggested` (top-level, all features)
- If `code_generation_activity_count` sums to 0, return `"N/A"` for acceptance rate
- Update the `CalculatedMetrics` interface to add `aiCodeAmplification: string`

### Step 2: Fix `DashboardMetrics.tsx`

- Rename "AI Suggestion Acceptance Rate" card to **"Acceptance Rate"**
- Update tooltip: "Percentage of Copilot code suggestions that developers actively accepted (event-based: acceptances / generations). If unavailable, data may not include event-level metrics."
- Rename "Code Generated by AI" to **"Total AI Code Added"**
- Update its tooltip: "Lines added to the editor through Copilot-related workflows, including accepted completions, applied code blocks, and agent/edit mode output."
- Add new KPI card: **"AI Code Amplification"** showing the ratio with tooltip: "Ratio of total code added vs code suggested. Values above 100% indicate agent/edit workflows adding code beyond ghost-text suggestions. This is NOT acceptance rate."

### Step 3: Fix `AcceptanceRateChart.tsx`

- Switch data source to event-based: aggregate `code_acceptance_activity_count` and `code_generation_activity_count` per period
- Rename title from "AI Adoption Quality Trend" to **"Acceptance Rate Trend"**
- Update helpText to: "Percentage of Copilot suggestions accepted by developers over time (event-based)."
- Remove `max: 100` constraint (event-based rate shouldn't exceed 100% but no artificial cap)
- Update tooltip label from "Acceptance Rate" to "Acceptance Rate"
- Handle zero generations: show 0% for periods with no generation events

### Step 4: Fix `ModelEffectivenessChart.tsx`

- Rename "Acceptance Rate" axis and series to **"AI Code Amplification (%)"**
- Remove `max: 100` on the y-axis (amplification can exceed 100%)
- Update helpText: "Compares AI Code Amplification (added/suggested ratio) and total interactions across AI models. This metric can exceed 100% because agent/edit workflows add code without corresponding suggested lines."
- Keep using `loc_added/loc_suggested` per model but label it correctly

### Step 5: Fix `UserProfileCard.tsx`

- Replace line-ratio "Acceptance Rate" with event-based: `SUM(code_acceptance_activity_count) / SUM(code_generation_activity_count) × 100`
- If generation count is 0, display "N/A"
- Keep label as "Acceptance Rate"

### Step 6: Fix `TopContributorsTable/dataProcessing.ts`

- Change `acceptanceRate` calculation to event-based: `stats.codeAcceptances / stats.codeGenerations × 100`
- Add `aiAmplification` field: `stats.acceptedLines / stats.suggestedLines` (existing line data)
- Handle division by zero: 0 if denominator is 0

### Step 7: Fix `TopContributorsTable/types.ts`

- Add `aiAmplification: number` to `ContributorWithSegment`
- Rename `acceptanceRate` label from "Acceptance Rate" to **"Acceptance Rate"** (keep name, fix formula)
- Add `aiAmplification` to `SortableColumn` and `columnLabels` as **"AI Amplification"**

### Step 8: Fix `TopContributorsTable/TopContributorsTable.tsx`

- Update help text: replace "Acceptance Rate = (Lines Added / Lines Suggested) × 100" with "Acceptance Rate = (Code Acceptances / Code Generations) × 100 (event-based)"
- Add "AI Amplification" column header
- Add note: "AI Amplification = Lines Added / Lines Suggested. Can exceed 100% due to agent/edit workflows."

### Step 9: Fix `TopContributorsTable/ContributorRow.tsx`

- Add cell for `aiAmplification` displaying `contributor.aiAmplification.toFixed(1)%`

### Step 10: Fix `TopContributorsTable/performanceSegments.tsx`

- Update `getPerformanceSegment` thresholds — acceptance rate will now be event-based (typically higher values since it's accept/generate events, not lines). Adjust thresholds:
  - Champion: acceptanceRate > 60 (was 40)
  - Producer: acceptanceRate > 40 (was 25)
  - Explorer: acceptanceRate > 20 (was 15)

### Step 11: Fix `CumulativeChart.tsx`

- Rename series: "Cumulative Lines Added" → **"Total AI Code Added (All Features)"**
- Rename series: "Cumulative Lines Suggested" → **"Code Suggested (Completions)"**
- Update helpText: "Running total of Copilot-related code added vs code suggested. Added lines include completions, chat, and agent output. Suggested lines only reflect ghost-text suggestions. Added can exceed suggested because agent/edit workflows add code without suggestions."

### Step 12: Add data explanation note

- Add a small info banner (or footer note) in `DashboardCharts.tsx` or `DashboardMetrics.tsx`:
  > "Line-based metrics represent Copilot suggestions and Copilot-related code added to the editor. They do not directly measure acceptance. Acceptance is only available via event-based metrics (Code Acceptances / Code Generations)."

### Step 13: Fix `CodeChurnChart.tsx` labels

- "Lines Added" tooltip/label → clarify as "AI Lines Added" or keep but ensure no "accepted" language
