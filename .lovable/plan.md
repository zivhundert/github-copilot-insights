

## Migration Plan: Cursor CSV to GitHub Copilot NDJSON

### GitHub Copilot Data Structure (from your file)

Each line is a JSON object per user per day with these key fields:

| Field | Description |
|-------|-------------|
| `day` | Date |
| `user_login` | Username (replaces Email) |
| `user_id` | Numeric user ID |
| `enterprise_id` | Organization ID |
| `user_initiated_interaction_count` | Total interactions started by user |
| `code_generation_activity_count` | Times code was generated |
| `code_acceptance_activity_count` | Times code was accepted |
| `loc_suggested_to_add_sum` | Lines of code suggested (add) |
| `loc_suggested_to_delete_sum` | Lines of code suggested (delete) |
| `loc_added_sum` | Lines actually added |
| `loc_deleted_sum` | Lines actually deleted |
| `used_agent` / `used_chat` / `used_cli` | Boolean feature flags |
| `totals_by_ide[]` | Breakdown by IDE (vscode, intellij, etc.) with plugin/IDE versions |
| `totals_by_feature[]` | Breakdown by feature (code_completion, chat_panel_agent_mode, chat_panel_ask_mode, agent_edit, chat_panel_custom_mode) |
| `totals_by_language_feature[]` | Per-language per-feature breakdown |
| `totals_by_language_model[]` | Per-language per-model breakdown |
| `totals_by_model_feature[]` | Per-model per-feature breakdown with interaction counts |

### Mapping: What stays, what changes, what's removed, what's new

#### Existing features that MAP directly (keep & adapt)

| Current Feature | GitHub Data Source | Notes |
|---|---|---|
| **Code Generated by AI** (metric) | `loc_added_sum` | Maps to accepted/added lines |
| **Acceptance Rate** (metric + chart) | `loc_added_sum / loc_suggested_to_add_sum` | Direct equivalent |
| **Hours Saved / Cost Savings / ROI** (metrics) | Same formula, use `loc_added_sum` | Works the same |
| **Active Users** (metric) | Count distinct `user_login` per period | No "Is Active" flag â€” anyone with a row is active |
| **Cumulative Chart** | `loc_added_sum` over time | Direct mapping |
| **Model Usage Chart** | `totals_by_model_feature[].model` | Richer data â€” multiple models per user per day |
| **Day of Week Chart** | `day` field | Same approach |
| **Top Contributors Table** | Aggregate per `user_login` | Adapt columns |
| **User Activity Chart** | `user_initiated_interaction_count` | Direct mapping |

#### Existing features to REMOVE (no equivalent data)

| Feature | Reason |
|---|---|
| **Cmd+K Usages** | Cursor-specific, no GitHub equivalent |
| **Bugbot Usages** | Cursor-specific |
| **Tabs Accepted / Tabs Shown** | No tab-specific metrics in GitHub data |
| **Tab Extension Word Cloud** | No "Most Used Tab Extension" field |
| **Apply Extension Word Cloud** | No "Most Used Apply Extension" field |
| **Programming Language Treemap** (current impl) | Currently uses "Most Used Apply Extension" â€” needs to be **rebuilt** using `totals_by_language_feature` instead |
| **Client Version Chart** (current impl) | Needs to be rebuilt â€” use `totals_by_ide[].last_known_ide_version` and `last_known_plugin_version` instead |
| **Subscription/API Key/Usage Based Reqs** | Cursor billing fields, no equivalent |

#### NEW features available from GitHub data

| New Feature | Data Source | Suggestion |
|---|---|---|
| **IDE Distribution Chart** | `totals_by_ide[].ide` | Pie/donut chart showing vscode vs intellij vs others |
| **Feature Usage Breakdown** | `totals_by_feature[].feature` | Stacked bar: code_completion, agent_mode, ask_mode, agent_edit, custom_mode |
| **Programming Language Treemap** (rebuilt) | `totals_by_language_feature[].language` | Actual programming languages (python, java, bash, etc.) |
| **Model Usage per Feature** | `totals_by_model_feature[]` | Which models are used for which features |
| **Agent/Chat/CLI Adoption** | `used_agent`, `used_chat`, `used_cli` | Track adoption of different modes over time |
| **Lines Deleted** | `loc_deleted_sum` | Track code deletions alongside additions |
| **Plugin Version Distribution** | `totals_by_ide[].last_known_plugin_version` | Track Copilot plugin versions across team |

### Implementation Steps

**1. Replace data model and parser**
- Replace `CursorDataRow` interface with `CopilotDataRow` matching the NDJSON structure
- Replace `csvParser.ts` with `ndjsonParser.ts` â€” parse line-by-line JSON
- Update `FileUpload.tsx` to accept `.ndjson` and `.json` files instead of `.csv`
- Update branding: "Cursor" references become "GitHub Copilot"

**2. Update data aggregation layer**
- Rewrite `dataAggregation.ts` for new fields
- Flatten nested arrays (totals_by_feature, totals_by_ide, etc.) during parsing
- User identification changes from `Email` to `user_login`

**3. Update metrics calculator**
- `loc_added_sum` replaces `Chat Accepted Lines Added`
- `loc_suggested_to_add_sum` replaces `Chat Suggested Lines Added`
- Remove references to Tabs, Cmd+K, Bugbot
- All users in the file are active (no "Is Active" filter needed)

**4. Adapt existing charts**
- **CumulativeChart**: Use `loc_added_sum`
- **AcceptanceRateChart**: Use `loc_added_sum / loc_suggested_to_add_sum`
- **ModelUsageChart**: Aggregate from `totals_by_model_feature` array (richer than before)
- **ChatRequestTypesChart** â†’ rename to **Feature Usage Chart**: Use `totals_by_feature` (code_completion, agent_mode, ask_mode, agent_edit)
- **DayOfWeekChart**: Use `day` field
- **TopContributorsTable**: Adapt columns, use `user_login`

**5. Remove Cursor-only charts**
- Remove `TabExtensionWordCloud`
- Remove `ApplyExtensionWordCloud`
- Remove Cmd+K and Bugbot from ChatRequestTypes

**6. Rebuild with new data**
- **ProgrammingLanguageTreemap**: Rebuild using `totals_by_language_feature[].language`
- **ClientVersionChart** â†’ **IDE/Plugin Version Chart**: Use `totals_by_ide` data

**7. Add new GitHub-specific charts**
- **IDE Distribution Chart**: Pie chart from `totals_by_ide`
- **Agent/Chat/CLI Adoption Chart**: Track boolean flags over time
- **Lines Added vs Deleted**: Dual-axis or stacked area chart

**8. Update settings and filters**
- Rename `cursorPricePerUser` to `copilotPricePerUser` (default: $19 for Business, $39 for Enterprise)
- Update `UserSelector` to use `user_login` instead of `Email`
- Update all tooltips and help text

**9. Update branding**
- Dashboard title and headers
- Upload instructions (link to GitHub Copilot admin export)
- Example showcase images
- File upload accept types (`.ndjson`, `.json`)
